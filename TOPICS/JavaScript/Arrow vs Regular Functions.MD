## ğŸ”¹ `this` Binding

This is the **most significant difference** between arrow and regular functions.

### Regular Function â€” `this` is dynamic

The value of `this` depends on **how** the function is called.

```js

  name: 'John',
  greet: function() {
    console.log(this.name);
  }
};

person.greet(); // John

const greetFn = person.greet;
greetFn(); // undefined (this now refers to global object or undefined in strict mode)
```

### Arrow Function â€” `this` is lexical

Arrow functions **inherit** `this` from their surrounding scope.

```js

  name: 'John',
  greet: () => {
    console.log(this.name);
  }
};

person.greet(); // undefined â€” arrow function takes 'this' from outside, not from person
```

But inside another function (nested scenario):

```js

  this.name = name;
  setTimeout(() => {
    console.log(`Hello ${this.name}`);
  }, 1000);
}

new Person('Alice'); // âœ… Hello Alice (works because arrow keeps 'this' from constructor)
```

Whereas using a regular function inside `setTimeout` would fail:

```js

  this.name = name;
  setTimeout(function() {
    console.log(`Hello ${this.name}`);
  }, 1000);
```
